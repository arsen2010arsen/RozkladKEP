# RozkladKEP Bot

👉 [Спробувати бота в Telegram](https://t.me/rozkladKEPbot)

Telegram-бот для автоматичного парсингу та розсилки розкладу занять Коледжу електронних приладів ІФНТУНГ (КЕП). Проєкт було повністю перезібрано для розширення функціоналу та перетворення його на персонального навчального асистента.

## Основний функціонал
* **Живий парсинг:** Отримання даних безпосередньо з офіційного сайту коледжу в реальному часі.
* **Автоматична розсилка:** Бот щодня о 00:00 автоматично надсилає актуальний розклад на наступний день усім підписаним користувачам.
* **Врахування тижнів:** Автоматичне визначення поточного тижня (чисельник/знаменник) відносно початку семестру.
* **Хмарна база даних:** Збереження профілів користувачів та їхніх налаштувань у MongoDB.

## Велике оновлення (v2.0) — Smart Assistant Update
В останній ітерації архітектуру бота було змінено для впровадження інтелектуальних функцій управління завданнями:

1. **Система персональних нотаток (FSM):**
   Впроваджено систему керування записами на базі скінченних автоматів (Finite State Machine). Це дозволяє надійно зберігати дедлайни та завдання, уникаючи помилок при введенні даних.

2. **Smart Subject Search (Розумний пошук):**
   Реалізовано алгоритм розпізнавання назв дисциплін за абревіатурами та ключовими словами (наприклад: ОПЗ, Укр літ). Система автоматично зіставляє ввід користувача з актуальним розкладом групи.

3. **Предметні папки:**
   Для оптимізації навігації реалізовано групування нотаток за навчальними дисциплінами. Користувач бачить структуру папок із предметами, що значно прискорює пошук потрібної інформації.

4. **Адаптивний UI:**
   Інтерфейс розкладу перероблено під сітку кнопок 3-2-1-4-1, що забезпечує швидкий доступ до всіх днів тижня та вибору груп на мобільних пристроях.

## Технологічний стек
* **Python 3.12+**
* **aiogram 3.x** — асинхронний фреймворк для Telegram API.
* **Motor (MongoDB)** — асинхронна робота з базою даних.
* **BeautifulSoup4 / BeautifulSoup** — веб-скрапінг розкладу.
* **APScheduler** — управління розкладом автоматичних розсилок.

## Налаштування та запуск
Бот оптимізований для деплою на платформі Render. Потрібні наступні змінні середовища:
* `BOT_TOKEN` — токен від @BotFather.
* `MONGO_URL` — рядок підключення до MongoDB Atlas.
* `ADMIN_ID` — цифровий ID адміністратора для доступу до статистики.
